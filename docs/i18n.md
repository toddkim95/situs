# i18n Maintenance

Situs keeps English as the canonical source for public documentation and
runtime strings.

## Runtime UI

Runtime CLI and picker text lives in `src/i18n/`. `mod.rs` owns locale
resolution, `I18n`, and `MessageKey`; each locale has its own table file such as
`en.rs`, `ko.rs`, and `zh_hans.rs`.

Supported runtime locales:

- `en`
- `ko`
- `zh-Hans`
- `es`
- `ja`

Locale resolution checks:

1. `SITUS_LANG`
2. `LC_ALL`
3. `LC_MESSAGES`
4. `LANG`
5. fallback `en`

Do not localize protocol or machine-readable values:

- command names and flags
- config keys
- environment variables
- paths
- history source IDs such as `local` and `atuin`
- widget protocol actions such as `run` and `cd`

## README Translations

The root `README.md` is the source of truth. Localized public READMEs live at:

- `docs/ko/README.md`
- `docs/zh-Hans/README.md`
- `docs/es/README.md`
- `docs/ja/README.md`

Each README should keep the same language switcher near the top:

```md
[English](...) | [한국어](...) | [简体中文](...) | [Español](...) | [日本語](...)
```

When the English README changes user-visible behavior, update the localized
READMEs in the same change or add an explicit note explaining why the localized
copy intentionally falls back to English.

Keep code blocks, commands, flags, environment variables, config keys, file
paths, and protocol words literal. Translate surrounding prose and table
descriptions.

## Verification

Run:

```sh
scripts/verify-doc-i18n.sh
```

This checks that the localized README files exist, that the root README links to
them, and that each localized README links back to English and the other
translations.

Runtime i18n is covered by Rust tests:

```sh
cargo test i18n
cargo test keymap_uses_korean_when_situs_lang_is_ko
cargo test keymap_uses_simplified_chinese_when_situs_lang_is_zh_hans
cargo test keymap_uses_spanish_when_situs_lang_is_es
cargo test keymap_uses_japanese_when_situs_lang_is_ja
```

CI runs `scripts/verify-doc-i18n.sh` in addition to the normal Rust checks.
Keep that script lightweight: it should catch missing localized README files,
language switcher drift, and broken cross-links without trying to fully judge
translation quality.

For picker text changes, also run the PTY smoke tests because terminal width and
wide-character rendering matter.

## Current Boundaries

The localized runtime surface currently covers the main interactive and
diagnostic paths: help, keymap, setup, doctor, stats, picker labels, picker help,
picker inspect text, and picker status messages.

Some low-level CLI errors and plain fallback prompts intentionally remain
English for now. When those messages become part of the polished public UX,
promote them into `MessageKey` and add acceptance coverage.
